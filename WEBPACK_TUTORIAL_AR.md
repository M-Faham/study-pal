# 🔧 Webpack & Bundlers Tutorial

## نظرة عامة

تم إضافة تيوتوريال شامل عن **Webpack والأدوات المشابهة** مع شرح مفصل بالعربية والإنجليزية.

هذا التيوتوريال يركز على **الشرح والفهم** بدون أسئلة - فقط دروس تعليمية مفصلة.

---

## 📚 محتوى الدروس

### 1️⃣ **ما هو Webpack؟**
- تعريف Webpack وأهميته
- لماذا نحتاج أدوات bundling؟
- كيف يعمل Webpack بشكل عام

### 2️⃣ **مفاهيم أساسية في Webpack**
- **Entry Point**: نقطة البداية
- **Output**: مكان الإخراج
- **Loaders**: معالجات الملفات
- **Plugins**: المكونات الإضافية
- مثال عملي لـ webpack.config.js

### 3️⃣ **أدوات بديلة ومشابهة**
شرح تفصيلي لـ:
- ⚡ **Vite** - الأسرع والأحدث
- 🎯 **Parcel** - Zero-config bundler
- 🦀 **esbuild** - مكتوب بـ Go
- 📦 **Rollup** - للمكتبات (libraries)
- 🔗 **Turbopack** - من فريق Next.js

مقارنة شاملة بين الأدوات

### 4️⃣ **كيف يعمل Bundling (التفاصيل الكاملة)**
- 5 مراحل للعملية:
  1. **Parsing** - تحليل الملفات
  2. **Dependency Resolution** - إيجاد التبعيات
  3. **Loading & Transformation** - معالجة الملفات
  4. **Bundling** - تجميع الملفات
  5. **Optimization** - تحسين النتائج
- شرح **Tree Shaking** (إزالة الكود الميت)
- مثال عملي كامل

### 5️⃣ **Development vs Production Build**
- وضع التطوير (Development):
  - ملفات كبيرة بدون ضغط
  - Source maps كاملة
  - Hot Module Reloading (HMR)
- وضع الإنتاج (Production):
  - ملفات مضغوطة وصغيرة
  - Code minification
  - Tree shaking

### 6️⃣ **أفضل الممارسات والنصائح**
7 نصائح عملية:
1. استخدام المسارات المطلقة (aliases)
2. Code Splitting (تقسيم الكود)
3. Lazy Loading للمكونات
4. Source Maps في الإنتاج
5. مراقبة حجم البundle
6. استخدام Caching
7. التحقق من الحجم في CI/CD
